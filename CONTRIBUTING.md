# Contributing to The Ultimate Competitive Programming Roadmap

First off, thank you for considering contributing to the roadmap! The community relies on eagle-eyed programmers like you to ensure this resource remains accurate, high-quality, and up-to-date.

## How Can I Contribute?

### Reporting Bugs & Broken Links
If you find a broken link, a problem that now requires a premium subscription, or an incorrectly verified problem title:
1. Go to the **Issues** tab on GitHub.
2. Click **New Issue**.
3. Clearly state the exact Tier, Section, and Problem Number that has an issue.
4. If possible, suggest a free and equivalent replacement problem from a platform like CSES, Codeforces, or SPOJ!

### Proposing Problem Replacements
If you believe a problem is misplaced (e.g., requires concepts far beyond its tier) or you have a significantly better pedagogical example for a topic:
1. Open an Issue outlining why the problem should be replaced.
2. If the maintainer approves, you can Fork the repository, make the change in the corresponding `tier_*.md` file, and submit a **Pull Request**.
3. **Important**: If you modify a `tier_*.md` file, please do **not** manually edit `ultimate_roadmap.md`. The ultimate roadmap is auto-generated by running `python scripts/build_roadmap.py`. Just submit your changes to the tier file, and the maintainer will compile it!

## Submitting a Pull Request
1. Fork the project.
2. Create your feature branch (`git checkout -b fix/broken-link`).
3. Commit your changes (`git commit -m 'Fix broken link for CSES 1650'`).
4. Push to the branch (`git push origin fix/broken-link`).
5. Open a Pull Request!
